Quickstart (what you’ll do)

  • Connect Archon as an MCP server (Local or Cloud).
  • Ask Claude to design your backend → Claude produces a DesignSpec.
  • Claude calls: validate_designspec → generate_project → docker_smoke.
  • You get a runnable repo with Swagger.
Recommended: start with Local MCP to pilot. Move to MCP Cloud for team onboarding + shared workspaces.

Local MCP (stdio)

Best for pilots and internal use. Runs on your machine; Claude connects via stdio.

Claude Desktop config (example)

{ "mcpServers": { "archon": { "command": "node", "args": ["./archon-mcp/dist/index.js"], "env": { "ARCHON_WORKSPACE": "/absolute/path/to/workspace" } } } }

Security: Archon should only write inside ARCHON_WORKSPACE.

Recommended local folder structure

/workspace /projects (generated repos go here) /specs (DesignSpec JSON files)

Workflow (what to ask Claude)

1) Create a DesignSpec

"Design a notifications backend. Use JWT + scopes. Entities: Notification, User. Include custom operation: /notifications/toggle. Use Redis cache and a queue for async delivery."

2) Validate spec

Tool: validate_designspec(spec) Expected: ok=true OR errors you fix in the spec.

3) Generate project

Tool: generate_project(spec, outDir) Output: runnable repo with Docker + Swagger.

4) Proof (Docker + Swagger)

Tool: docker_smoke(outDir) Expected: containers healthy + /health ready + Swagger loads

5) Evolve safely

  • inject_module — add Redis/queue/payments
  • add_entity — add new domain entities incrementally

Troubleshooting

Claude can’t connect to MCP

  • Check Claude Desktop config JSON syntax.
  • Verify the server command path exists and runs locally.
  • Confirm Node version is installed and accessible.

Validation errors

  • Fix schema errors first (missing required fields).
  • Then fix semantic errors (e.g., invalid scopes, missing entity binding).

Docker smoke fails

  • Ensure Docker is installed and running.
  • Check .env values (DB URL, Redis URL if injected).
  • Run compose manually and inspect logs.
